## 2. Files to Create

### 2.1 `src/dream/cluster/types.ts`

```
interface MemoryCluster {
  id: string
  level: ClusterLevel
  members: string[]
  confidence: number
  metadata: Record<string, unknown>
}

type ClusterLevel = "topic" | "claim" | "conflict"
```

Pure type definitions. No I/O, no logic.

### 2.2 `src/dream/cluster/conflict-adapter.ts`

```
class ConflictClusterAdapter {
  fromGraph(graph: ConflictGraph): MemoryCluster[]
}
```

Rules:
- 每个连通分量 → 一个 `MemoryCluster`（level="conflict"）
- 空图 → `[]`
- 1-node 分量 → 不输出（至少 2 members）
- `id` = `conflict-${componentIndex}`
- `metadata.conflictIds` = 原始冲突边 ID 列表
- 不调用任何 LLM，不重新检测冲突

### 2.3 `src/dream/analyzer/evidence.ts`

```
interface EvidenceReport {
  clusterId: string
  members: Array<{
    filename: string
    worthiness: number
    confidence: number
    usage: number
    feedback: number
    temporal: number
  }>
  dominant?: string
}

class EvidenceCollector {
  collect(cluster: MemoryCluster, memories: MemoryFile[]): EvidenceReport
}
```

Rules:
- `worthiness` = 从 `WorthinessScore` 读取（平均值）
- `confidence` = 从 `MemoryProvenance.confidence` 映射（explicit=1.0, inferred=0.6, uncertain=0.3）
- `usage` = 从 `RecallEvent` 统计（未来 v0.5.2 做，v0.5.0 先设为 0）
- `feedback` = 从 `RecallQualityMetrics` 读取（未来 v0.5.2，先设为 0）
- `temporal` = 从 `temporalValidity()` 计算
- `dominant` = 如果有成员在所有维度上显著高于其他成员，标记文件名

### 2.4 `src/dream/candidate/types.ts`

```
interface ConsolidationCandidate {
  id: string
  clusterId: string
  action: CandidateAction
  targets: string[]
  evidence: EvidenceReport
  expectedGain: number
  riskHint: number
  confidence: number
}

type CandidateAction = "merge" | "archive" | "split" | "keep"
```

Pure type definitions.

### 2.5 `src/dream/candidate/generator.ts`

```
class CandidateGenerator {
  generate(clusters: MemoryCluster[], evidence: EvidenceReport[]): ConsolidationCandidate[]
}
```

For conflict clusters specifically:
- 如果 cluster 有 dominant member 且其他成员低 worthiness (<0.2) → `archive`（低风险）
- 如果 cluster 所有成员 worthiness 均低 → `archive`（低风险）
- 如果 cluster 无 dominant、无低 worthiness → `keep`（不生成 candidate）
- 其他 case → `split`（标记冲突）

### 2.6 `src/dream/mode.ts`

```
export type DreamMode = "classic" | "v0.5" | "hybrid"
```

Enum only. No behavioral change — existing `dream.ts` unchanged.
