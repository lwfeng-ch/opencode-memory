## Sprint 2: Conflict Resolution

### Task 4: Backend — Conflict API

**Files**: `src/server/routes/conflicts.ts`

**Endpoints**:
- `GET /api/v1/conflicts` — list all conflicts
- `GET /api/v1/conflicts/:id` — single conflict detail
- `POST /api/v1/conflicts/:id/resolve` — resolve conflict

**Conflict model**:
```typescript
{
  id: string;
  claimA: { text: string; timestamp: string; source: string };
  claimB: { text: string; timestamp: string; source: string };
  resolution: { strategy: string; confidence: number; explanation: string };
  status: "open" | "resolved" | "dismissed";
}
```

### Task 5: Frontend — Conflict components + page

**Components**:
- `ConflictCard.tsx` — compact conflict display with claims
- `ConflictPanel.tsx` — full detail with AI explanation

**Page**: `memory-console/src/app/conflicts/page.tsx`
